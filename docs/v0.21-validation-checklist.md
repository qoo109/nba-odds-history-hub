# V0.21 validation checklist

The pull request is acceptable only when all checks below pass without changing the disabled boundary.

- Contract schema and request state are exact.
- Approval requested, recorded and granted remain false.
- Maximum execution count remains zero.
- The committed preseason fixture path, filename, bytes, SHA-256, schema and season match exactly.
- The temporary workspace is outside the repository.
- Seed, mutation and restored aggregate SQLite counts match the contract.
- The seed database and backup bytes match.
- Mutation changes the temporary database.
- Restore returns the database to the backup SHA-256 and seed counts.
- Canonical events, raw imports and odds snapshots remain zero.
- No external schedule, production database, network, recurring collection or cross-repository write is used.
- No row-level artifact, shell command or production implementation module is emitted.
- Full repository tests and the focused workflow pass.

A green workflow proves execution of these checks only. It does not grant production approval.
