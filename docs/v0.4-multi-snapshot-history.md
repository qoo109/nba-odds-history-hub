# V0.4 Multi-snapshot History Builder

## Purpose

V0.4 converts normalized SQLite rows into stable quote histories while preserving the project's point-in-time and provenance boundaries.

## Quote identity

The history key includes:

- source
- bookmaker
- source event
- market type
- period
- side
- participant
- alternate-market flag

Price and line are values, not identity fields. This allows a line or price change to remain in the same history.

## Movement activation rule

A quote is movement-ready only when it has at least two **distinct** `observed_at` values.

- One timestamp: stored and visible, but no movement claim
- Two or more timestamps: first/latest comparison and chart eligible
- Two rows at the same timestamp: retained and flagged as duplicate observation time

## Outputs

### `odds_history_grouped.json`

Contains coverage, readiness flags, quote identity, observations, first/latest prices, and movement deltas.

### `source_health.json`

Summarizes accepted rows, snapshot times, source events, sources, and bookmakers. It never interprets an absent row as an unchanged quote.

### `nba_value_lab_odds_export.json`

Provides a stable research contract containing source IDs, bookmaker IDs, source/canonical event IDs, market identity, price, line, and `observed_at`.

The export deliberately keeps these gates false:

- opening/closing classified
- executable backtest ready
- CLV ready
- EV/ROI ready
