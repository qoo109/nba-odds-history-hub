# V0.3 History Quality Controls

Updated: 2026-07-20

## Scope

This slice upgrades the manual importer from “can store a snapshot” to “can explain what was accepted, what was missing, and whether a quote actually changed.”

## Added controls

### Import quality report

Every CLI import writes `import_quality_report.json` with:

- matched `matchupId` values
- market rows whose `matchupId` is missing from `matchups.json`
- matchup rows that have no straight market
- duplicate matchup IDs
- duplicate market keys
- duplicate participant IDs inside one market
- orphan participant prices
- invalid price-entry counts
- raw combined SHA-256

The report is structural QA. It does not prove the source is complete, licensed for redistribution, or suitable for market backtesting.

### Change-aware history retention

`--dedupe-mode changes-only` stores a row only when its line or American price differs from the latest earlier observation for the same quote identity.

Quote identity uses:

```text
source + bookmaker + source_event_id + market_type + period
+ side + participant_id + alternate flag
```

The comparison is strictly against observations earlier than the current `observed_at`. A later snapshot is never used to decide whether an older backfill row changed.

`--dedupe-mode exact` remains available and preserves every distinct timestamped quote.

### Source and bookmaker registries

SQLite now contains:

- `data_sources`
- `bookmakers`
- `source_events`
- `canonical_events`

`source_events` are created as `unmapped` placeholders. The system does not invent NBA Value Lab event IDs. A source event becomes `mapped` only after an explicit canonical mapping is written.

### V0.2 database migration

Existing V0.2 SQLite files are migrated additively with:

- `bookmaker_id`
- `canonical_event_id`
- registry tables
- quote-history index

No historical row is rewritten as opening, closing, or line movement.

## Boundaries

- One observation time is still not a movement history.
- `cutoff_at` is not `observed_at`.
- Change-aware storage is not CLV or ROI calculation.
- Source-event placeholders are not verified NBA Value Lab joins.
- Missing rows remain missing; they are never treated as unchanged prices.
